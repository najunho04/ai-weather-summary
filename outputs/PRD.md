# AI 날씨 요약 PRD

## Executive Summary

"AI 날씨 요약"은 공공데이터 기상청 단기예보 API의 날씨 데이터를 Anthropic Claude API로 추론·요약하여, 일반 개인 사용자가 주요 도시의 날씨를 읽기 쉬운 자연어 한 단락으로 확인할 수 있게 해 주는 간단한 웹사이트다. 사용자는 도시를 선택하기만 하면 숫자 표 대신 "오늘 오후부터 비가 오니 우산을 챙기세요" 같은 요약을 받는다. 주요 사용자는 빠르게 날씨를 확인하려는 일반 개인 사용자다.

## Problem Statement

일반 사용자가 기상청 원본 예보 데이터를 직접 보면, 시간대별 기온·강수확률·하늘상태·풍속 등 수십 개의 숫자 코드가 표 형태로 나열되어 있어 "지금 우산을 챙겨야 하는가" 같은 단순한 판단조차 빠르게 내리기 어렵다. 기상청 단기예보 API는 SKY, PTY 같은 코드값과 격자좌표(nx, ny)를 사용하기 때문에 일반 사용자가 그대로 해석하기에는 진입 장벽이 높다.

본 프로젝트는 이 원본 데이터를 서버에서 받아 Claude API에 전달하고, 사람이 읽기 좋은 자연어 요약 한 단락으로 변환해 화면에 보여 줌으로써 "숫자 해석" 부담을 없앤다. 복잡한 예보 화면이나 장기 예보가 아니라, 선택한 도시의 단기 날씨를 한눈에 이해시키는 것이 정확한 해결 범위다.

## Target Users

### 날씨를 빠르게 확인하려는 개인 사용자

출근·등교·외출 전에 오늘과 내일 날씨를 빠르게 확인하려는 일반 개인 사용자다. 기상 전문 지식이 없고, 코드값이나 시간대별 표를 해석할 의향이 없으며, "비가 오는지", "더운지", "우산이 필요한지" 같은 결론만 알고 싶어 한다. 주로 도시 선택과 AI 요약 결과 확인 기능을 사용한다.

## User Stories

- US-001: As a 일반 개인 사용자, I want 주요 도시를 드롭다운에서 선택하기를 원한다 so that 격자좌표를 몰라도 내 지역 날씨를 조회할 수 있다.
- US-002: As a 일반 개인 사용자, I want 선택한 도시의 날씨를 자연어 요약 한 단락으로 보기를 원한다 so that 숫자 표를 해석하지 않고도 오늘 날씨를 바로 이해할 수 있다.
- US-003: As a 일반 개인 사용자, I want 우산·겉옷 등 행동 권고가 포함된 요약을 보기를 원한다 so that 외출 준비를 빠르게 결정할 수 있다.
- US-004: As a 일반 개인 사용자, I want 데이터 조회나 요약이 실패했을 때 명확한 안내 메시지를 보기를 원한다 so that 빈 화면 앞에서 혼란스럽지 않게 다시 시도할 수 있다.

## Functional Requirements

### FR-001: 주요 도시 드롭다운 선택
사용자가 미리 정의된 주요 도시 목록에서 하나를 선택하면, 해당 도시가 기상청 단기예보 API용 격자좌표(nx, ny)로 매핑된다.

**Acceptance Criteria:**
- 화면에 최소 5개 이상의 주요 도시(예: 서울, 부산, 대구, 인천, 광주)가 드롭다운으로 표시된다.
- 도시 선택 시 내장된 도시→격자좌표(nx, ny) 매핑 테이블에서 좌표가 결정된다.
- 도시를 선택하지 않은 상태에서 조회 버튼을 누르면 도시 선택 안내가 표시된다.

### FR-002: 기상청 단기예보 데이터 조회
선택한 도시의 격자좌표로 기상청 단기예보 API(VilageFcstInfoService_2.0)를 서버에서 호출하여 단기 날씨 데이터를 가져온다.

**Acceptance Criteria:**
- 백엔드가 nx, ny와 가장 최근 발표시각(base_date, base_time)으로 단기예보 API를 호출한다.
- 응답에서 기온(TMP), 하늘상태(SKY), 강수형태(PTY), 강수확률(POP) 등 핵심 항목을 추출한다.
- API 응답이 오류이거나 비어 있으면 사용자에게 조회 실패 메시지를 반환한다.

### FR-003: Claude API 기반 날씨 추론 및 요약 생성
조회된 단기예보 데이터를 Anthropic Claude API로 전달하여, 사람이 읽기 좋은 자연어 요약 한 단락을 생성한다.

**Acceptance Criteria:**
- 백엔드가 추출한 날씨 데이터를 프롬프트로 구성하여 Claude API를 호출한다.
- 응답에는 기온 추이, 강수 여부, 우산·겉옷 등 행동 권고가 포함된 한국어 자연어 요약이 담긴다.
- Claude API 호출이 실패하면 요약 생성 실패 메시지를 반환한다.

### FR-004: AI 날씨 요약 결과 출력
생성된 자연어 요약을 선택한 도시 이름과 함께 웹 화면에 보여 준다.

**Acceptance Criteria:**
- 조회 완료 후 도시 이름과 AI 요약 텍스트가 화면에 표시된다.
- 데이터를 가져오는 동안 로딩 상태가 표시된다.
- 조회 또는 요약 단계에서 오류가 발생하면 사용자에게 명확한 오류 메시지가 표시된다.

## Non-Functional Requirements

- NFR-001: 프론트엔드는 순수 HTML/JS로 구현하고 백엔드는 Node.js(Express)로 구현하여 별도 빌드 도구 없이 실행할 수 있어야 한다.
- NFR-002: 외부 연동은 공공데이터 기상청 단기예보 API(VilageFcstInfoService_2.0)와 Anthropic Claude API만 사용한다.
- NFR-003: 공공데이터 인증키와 Claude API 키 등 모든 API 키는 서버 환경변수로만 보관하며, 브라우저(클라이언트) 코드에 노출하지 않는다.
- NFR-004: 도시 선택부터 요약 출력까지의 전체 응답 시간은 정상 네트워크 환경에서 95-percentile 10초 이내를 목표로 한다.
- NFR-005: 처리 범위는 단기예보(오늘~최대 3일)로 한정하며, 그 외 기간 예보는 호출하지 않는다.

## Out of Scope

- 회원가입·로그인 (인증 없이 누구나 바로 사용)
- 다국어 지원 (한국어 UI만 제공)
- 중기예보·주간예보 (단기예보만 처리)
- 날씨 데이터 DB 저장 및 이력 조회 (매 요청 시 실시간 조회만 수행)

## Success Metrics

- 도시 선택부터 AI 요약 출력까지 정상 케이스 응답 시간 95-percentile 10초 이내
- 정상 네트워크 환경에서 날씨 조회 + 요약 생성 성공률 95% 이상
- 단기예보 API 호출 1회당 사용자에게 노출되는 원본 코드값(SKY, PTY 등) 0개 (모두 자연어로 변환)
- 오류 발생 시 사용자에게 안내 메시지가 표시되는 비율 100%
